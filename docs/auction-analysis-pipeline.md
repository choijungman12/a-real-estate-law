# 경매 권리분석·시장가치분석 파이프라인 설계

**작성일**: 2026-04-25
**전략**: 지지옥션 유료회원 권한으로 다운로드한 매물 정보(PDF/텍스트)를 입력 → 자체 LLM·법령 API·실거래가 API로 재분석 → 자체 UI로 출력

---

## 1. 데이터 입력 — 지지옥션 자료 활용 방식

### 1-1. 사용자가 업로드하는 자료
| 자료 | 형식 | 핵심 추출 항목 |
|---|---|---|
| 경매물건 상세서 | PDF | 사건번호, 소재지, 감정가, 최저가, 입찰일, 용도, 면적 |
| 권리분석 보고서 | PDF/텍스트 | 말소기준권리, 인수/소멸 권리, 임차인 현황, 배당순위 |
| 등기부등본 요약 | PDF | 갑구·을구 권리관계, 근저당, 가압류, 가처분 |
| 임차인 현황표 | PDF/표 | 전입일, 확정일자, 보증금, 대항력 여부 |
| 매각물건명세서 | PDF | 법원 공식 권리 안내 |
| 현황조사서 | PDF | 점유관계, 명도 난이도 단서 |

### 1-2. 처리 단계
```
[지지옥션 PDF 업로드]
        ↓
[PDF 파서] (pdfplumber/pypdf or Claude PDF native input)
        ↓
[구조화 추출] (LLM이 항목별 JSON 생성)
        ↓
[검증·재분석 파이프라인] (아래 2장)
        ↓
[자체 권리분석·시장가치 보고서] (UI 출력)
```

### 1-3. PDF 파싱 옵션 비교
- **Claude API 네이티브 PDF 입력** (권장) — Sonnet 4.6/Opus 4.7가 PDF를 직접 읽고 표·이미지 인식. 별도 OCR 불필요.
- **Python pdfplumber** — 표 추출 정확. LLM 비용 절감용.
- **Tesseract OCR** — 스캔 PDF 폴백.

## 2. 권리분석 재검증 파이프라인

지지옥션 분석을 그대로 신뢰하지 않고, **공식 법령·실거래가로 교차검증**한 자체 결과를 생성합니다.

### 2-1. 입력 정규화 (Schema)
```typescript
type AuctionCase = {
  caseNumber: string;        // 2025타경12345
  court: string;
  address: { road: string; jibun: string; sido: string; sigungu: string; dong: string };
  propertyType: '아파트' | '오피스텔' | '상가' | '토지' | '단독' | '공장' | '지식산업센터';
  area: { land: number; building: number };
  appraisalPrice: number;
  minimumBid: number;
  bidDate: string;
  rights: Right[];           // 등기부 권리 (시간순)
  tenants: Tenant[];         // 임차인 현황
  notices: string[];         // 매각물건명세서 특이사항
};

type Right = {
  type: '근저당' | '저당' | '가압류' | '압류' | '가처분' | '가등기' | '전세권' | '임차권' | '지상권' | '예고등기';
  date: string;
  amount?: number;
  holder: string;
  isCancellationBase?: boolean;  // 말소기준권리 여부
  willSurvive: boolean;          // 인수 여부 (낙찰자 부담)
};

type Tenant = {
  name: string;
  moveInDate: string;        // 전입일
  fixedDate: string;         // 확정일자
  deposit: number;
  monthlyRent?: number;
  hasOpposingPower: boolean; // 대항력
  hasPriorityPayment: boolean; // 우선변제권
};
```

### 2-2. LLM 권리분석 모듈 (Claude Sonnet 4.6)

**입력**: 정규화된 `AuctionCase` + 국가법령정보 API에서 가져온 관련 법령 조문

**검증 항목**:
1. **말소기준권리 판단** — 최선순위 (근)저당·압류·담보가등기·경매개시결정등기 중 가장 빠른 것
2. **인수/소멸 권리 분류** — 말소기준권리보다 빠른 권리는 낙찰자 인수
3. **대항력 임차인 검출** — 전입일 < 말소기준권리일 && 점유 확인
4. **우선변제권 검출** — 대항력 + 확정일자
5. **소액임차인 최우선변제** — 주택임대차보호법 시행령 기준 (지역별 한도)
6. **유치권·법정지상권 위험 신호** — 매각물건명세서·현황조사서 텍스트 분석
7. **위반건축물·NPL·체납 단서** — 노출된 메모 분석

**출력**:
```json
{
  "cancellationBase": { "type": "근저당", "date": "2020-03-15", "holder": "..." },
  "surviving_rights": [...],     // 인수 권리
  "extinguished_rights": [...],  // 소멸 권리
  "tenant_risks": [
    { "name": "홍길동", "risk": "대항력+확정일자, 보증금 1억 인수 가능성" }
  ],
  "warnings": [
    "현황조사서에 '점유자 폐문부재' → 명도 난이도 상",
    "매각물건명세서 특이사항: 유치권 신고액 2,000만원 — 진위 검토 필요"
  ],
  "confidence": 0.85,
  "discrepancies_with_jijiauction": [
    "지지옥션은 임차인 A를 대항력 없음으로 판단했으나, 전입일 재확인 시 말소기준권리보다 1일 빠름 → 재검토 필요"
  ]
}
```

### 2-3. 법령 교차검증 (RAG)
- **국가법령정보 OPEN API**로 다음 법령을 RAG 인덱스에 적재:
  - 민사집행법 (경매 절차)
  - 주택임대차보호법 / 시행령 (소액임차인 한도)
  - 상가건물임대차보호법
  - 민법 제320조~328조 (유치권)
  - 가등기담보 등에 관한 법률
  - 국세징수법 (조세 우선)
- 각 권리 판단 시 LLM이 **인용한 조문 번호**를 함께 출력

### 2-4. 시장가치분석 모듈
**데이터 소스**:
- 국토부 실거래가 API (반경 1km, 최근 12개월, 동일 평형 ±10%)
- R-ONE 지수 (지역별 가격지수 트렌드)
- 카카오맵 거리·교통 분석
- (선택) 학군·역세권 데이터

**계산**:
```
시장가치 = median(인근 실거래가) × 평형보정 × 층수보정 × 경과월수보정
적정 입찰가 = 시장가치 × (1 - 경매할인율) - 인수권리 합계 - 명도비용 추정
수익률 = (시장가치 - 입찰가 - 인수권리 - 세금 - 명도비) / 투자금
```

**출력**:
- 인근 실거래가 분포 차트
- 감정가 vs 시장가 vs 최저가 vs 추정 적정 입찰가
- 시나리오 (보수/중립/공격적 입찰가)
- 예상 수익률 (단기 매도, 임대 수익형 분리)

## 3. 자체 사이트 구성 (지지옥션 벤치마킹)

### 3-1. 핵심 화면
| 화면 | 기능 |
|---|---|
| 매물 업로드 | 지지옥션 PDF 드래그&드롭, 자동 파싱 |
| 매물 상세 | 정규화된 정보 + 지도(카카오/V월드 3D) + 인근 실거래가 |
| 권리분석 | 자체 LLM 분석 + 지지옥션 결과 비교 + 법령 인용 |
| 시장가치 | 차트, 입찰가 시나리오, 수익률 시뮬레이션 |
| 명도/대출 | 명도 난이도 + 경락잔금대출 한도 추정 |
| 즐겨찾기·알림 | 입찰일 임박 알림 (텔레그램) |

### 3-2. 데이터 흐름
```
지지옥션 PDF
  ↓ (사용자 업로드)
S3/Storage 저장
  ↓
PDF 파서 (Claude PDF input)
  ↓
AuctionCase JSON
  ↓
┌────────────────┬─────────────────┬─────────────┐
권리분석 LLM     실거래가 조회     V월드 지적     국가법령 RAG
  ↓               ↓                 ↓               ↓
└────────────────┴─────────────────┴─────────────┘
  ↓
통합 보고서 (PostgreSQL 저장)
  ↓
Next.js UI 렌더링
```

### 3-4. 법적·약관 주의사항
- **지지옥션 자료 재배포 금지**: 사용자 본인 계정으로 받은 PDF는 **본인이 본인 계정에서 분석에만 사용** — 시스템에 저장하더라도 사용자 본인 계정 영역에 격리.
- **타 사용자에게 노출/공유 기능 제공 금지** (유료 회원 약관 위반 가능).
- **공식 출처 데이터(국토부·법원·법령정보)는 자유롭게 가공·표시 가능**.
- 사용자에게 약관 동의 화면에서 명시: "본 서비스는 사용자가 합법적으로 취득한 자료만 분석하며, 제3자 재배포·공유는 금지됩니다."
